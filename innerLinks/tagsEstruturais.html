Tipos de Tags Semânticas

Existem 2 tipos de tags semânticas: as Estruturais e as do Nível de Texto. As estruturais servem para estruturar a página visualmente, enquanto as do Nível do Texto para alterar visualmente um texto dentro da página.

Ir para as Tags do Nível do Texto

Tags Estruturais

<h1> - <h6>

São usados para escrever um título para uma parte do site. Eles vão de <h1> até <h6>, e o número após o “h” define o tamanho e importância do título (sendo h1 maior e h6 o menor).

É importante citar que h1 deve ser usado Código uma vez no texto inteiro, descrevendo o título da página inteira. E também que não se deve “pular” uma das sequências do h: primeiro use h1, depois h2, depois h3 e assim por diante, afinal, eles são organizados por ordem de importância também.

<form>

São usados para construir formulários. Dentro dela, você pode por outras tags, como por exemplo a tag <input>, para criar blocos onde o usuário pode fornecer dados para o site.

A propiedade “action” dessa tag se refere ao destino das informações dadas pelo usuário após ele clicar em um <input> do tipo “submit”.

<table>

São usados para construir tabelas, com quantia de colunas e fileiras customizáveis.

É importante recitar que nessa tag, você pode definir uma nova linha com , um título com <th> e uma coluna com <td>, que precisa estar necessariamente dentro de um <tr>. Caso o <th> esteja fora de um <tr>, ele vai ter a largura da tabela inteira, servindo como um título geral para a mesma.

<address>

É usada para definir informações de contato do autor de um documento ou artigo. É onde anexamos e-mails e números de celular por exemplo.

<header>

Ela é usada como um contêiner para a introdução de um conteúdo. Ela geralmente contém:

  • Um ou mais dos elementos h1-h6;
  • Informações de autoria da página;
  • Um logo/ícone.

<section>

É usada para indicar uma seção independente do conteúdo do site, o dividindo em diferentes “blocos” lógicos.

Esta não deve também ser usada para substituir a tag <div> completamente, como na criação da estrutura de uma página, onde devemos usar <div> no lugar de <section>.

<article>

É usada para indicar um bloco de artigo no site. Ela deve conter pelo menos:

  • Um <p>;
  • Um <header>;

<details>

É usada para criar um bloco de texto interativo com um título, onde o usuário pode comprimir e extender o texto clicando no título para otimizar o espaço ocupado pelo texto.

<summary>

É usada para definir um título a tag <details>, que pode ser clicada para abrir ou fechar a aba do texto.

Usamos as mesmas imagens do tópico <details> pois <summary> sempre vai vir acompanhada dessa outra, como no exemplo acima, que o texto Epcot Center é o título que aparece mesmo quando o texto está comprimido.

<nav>

É usado para agrupar em um bloco uma aba com vários links de navegação no mesmo lugar. Nele, a tag <a> é obrigatória, mas você pode escrever textos sem link nela também

<main>

Ele especifica qual o conteúdo mais importante/principal do site inteiro. Ele deve ser usado Código uma vez, e não pode estar dentro das tags:

  • <article>;
  • <aside>;
  • <footer>;
  • <header>;
  • <nav>;

É importante citar também, que essa tag não só pode, mas muitas vezes é encontrada dentro da tag <section> e/ou <div>.

<aside>

É usada para indicar um conteúdo secundário ao texto principal, encontrado na tag <main>. Este deve estar relacionado indiretamente ao mesmo.

<figure> & <figcaption>

A tag figure é usada para ilustrar uma figura dentro do site, como ilustrações, diagramas, fotos, listas de codigo, entre varias outras. Enquanto figcaption é usada para inserir uma legenda a tag <figure>.

<footer>

A tag Footer é usada para criar rodapés dentro do site, geralmente contendo:

  • Informações de autoria;
  • Informações de Direitos Autorais;
  • Informações de contato (e-mails, telefone, redes sociais, etc.)
  • Um sitemap (informações sobre a página, links de arquivos e sites relacionados, etc.);
  • Links para voltar ao topo da página
  • Documentos relacionados ao site
